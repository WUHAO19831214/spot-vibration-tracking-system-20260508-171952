# 光斑振动追踪系统

**Spot Vibration Tracking System**

在线演示：https://wuhao19831214.github.io/spot-vibration-tracking-system-20260508-171952/

## 项目解决什么问题

本项目服务于受迫振动、周期振动和频率响应等物理实验。它使用浏览器摄像头追踪白纸上的红色反射光斑，在每个驱动力频率停留期间计算光斑纵向位置的最大值与最小值，并绘制“光斑振动幅度—驱动力频率”关系。

项目主要改善人工逐帧查看视频、手工标点和抄录频率数据时效率低、读数主观、时间同步困难的问题。

## 测量对象与统一术语

系统的**直接测量对象是摄像头图像中红色光斑重心的纵坐标** `y`。每个频率停留窗口内：

```text
Δy_pixel = y_max - y_min
Δy_cm = Δy_pixel × cm_per_pixel
```

文档统一称其为：

- **光斑振动幅度 Δy / cm**；或
- **光斑上下位移范围 Δy / cm**。

这里的“幅度”是光斑在统计窗口内的**峰—峰位移范围**，不是严格意义上“平衡位置到最大位移”的机械振幅 `A`。系统没有根据反射光路放大率反演钢尺或振动物体本体的真实振幅。若要得到本体振幅，必须另行建立并标定光路几何模型。

## 输入是什么

当前输入包括：

1. 浏览器通过 `getUserMedia()` 获取的**实时摄像头视频流**；
2. 用户在视频画面上绘制的一段实际长度为 1 cm 的标尺；
3. Web Audio API 产生的正弦波驱动频率参数；
4. 自动扫频设置：起始频率、峰值频率、步进 `Δf`、停留时间 `Δt`。

当前版本**不支持**上传静态图片、预录视频或外部传感器文件。

## 红色光斑识别算法

程序逐帧把视频绘制到隐藏 Canvas，并读取 RGB 像素。像素经 RGB→HSV 转换后，满足下列条件时被视为红色候选：

- 色相 `h ≤ 18°` 或 `h ≥ 340°`；
- 饱和度 `s > 0.38`；
- 明度 `v > 0.35`；
- `r > 135`、`r-g > 35`、`r-b > 20`。

候选像素按饱和度、明度和红色优势加权，计算加权重心：

```text
x_c = Σ(x_i w_i) / Σw_i
y_c = Σ(y_i w_i) / Σw_i
```

当总权重大于内部阈值 900 时，当前帧被视为锁定光斑。视频宽度超过 1000 px 时按 2 px 步长抽样以降低计算量。这是颜色阈值启发式算法，不是神经网络或 OpenCV 目标检测器。

## 像素如何转换为厘米

用户点击“绘制 1cm 标尺”，在光斑所在平面沿已知 1 cm 长度拖出线段。系统将屏幕坐标映射回原始视频坐标，得到像素长度 `L_pixel`：

```text
cm_per_pixel = 1 / L_pixel
Δy_cm = Δy_pixel × cm_per_pixel
```

标尺应与光斑所在平面共面。透视、景深、镜头畸变和相机移动都会使比例失真。

## 输出哪些量

自动扫频期间，每个唯一频率最多保留一条记录：

| 字段 | 含义 | 单位 |
|---|---|---|
| `index` | 记录序号 | — |
| `frequency` | Web Audio 当前设定频率 `f` | Hz |
| `spot_displacement_range` | 光斑纵坐标范围 `Δy=y_max-y_min` | cm |
| `time` | 自页面初始化后的记录时间 | s |

往返扫频再次经过同一频率时，程序不新增重复行，而保留较大的 `Δy`。图表以频率为横轴、光斑振动幅度为纵轴绘制散点，并以三次贝塞尔曲线连接。连接曲线是视觉呈现，不是理论模型拟合。

## 如何运行

### 在线运行

打开在线演示并允许摄像头权限。摄像头访问要求 HTTPS 或 localhost 安全上下文。

### 本地运行

```bash
git clone https://github.com/WUHAO19831214/spot-vibration-tracking-system-20260508-171952.git
cd spot-vibration-tracking-system-20260508-171952
python3 -m http.server 4173
```

访问 `http://localhost:4173/`。不要直接双击 `index.html`，部分浏览器会阻止 `file://` 页面访问摄像头。

### 推荐实验流程

1. 固定摄像头、白纸和反射光路；
2. 清除背景中的红色干扰物；
3. 打开摄像头并确认光斑稳定锁定；
4. 在光斑平面完成 1 cm 标尺标定；
5. 设置扫频范围、步进和每点停留时间；
6. 启动音频输出和自动扫频；
7. 检查每个频率仅生成一条 `Δy`；
8. 保存实验条件、截图和结果。

## 典型实验结果

[示例数据](examples/sample_sweep_data.csv) 是**合成格式示例**，不是经过溯源校准的正式实验结果。典型趋势可能表现为低频区 `Δy` 较小、某一频段增大、随后减小。曲线峰值只能作为候选响应峰；不能仅凭单次光斑曲线认定振动物体本体的共振参数。

正式论文数据应同时提供装置结构、标定证据、摄像头设置、原始视频、重复次数和不确定度。

## 误差来源

- 背景红色物体、反光或过曝造成误识别；
- 光斑散斑、拖影和中心发白导致重心偏移；
- 摄像头自动曝光、自动对焦、滚动快门和帧率不足；
- 频率切换瞬态被计入停留窗口；
- 标尺与光斑不共面、透视和镜头畸变；
- 反射光路角度放大未建模；
- 扬声器或执行器实际输出与设定值不一致；
- `setInterval` 和 `requestAnimationFrame` 调度抖动；
- 当前没有滤波、离群点剔除或置信区间。

详见 [已知局限](docs/KNOWN_LIMITATIONS.md)。

## 相比人工视频分析的改进

- 实时自动定位光斑，减少逐帧手工标点；
- 自动关联频率与位移范围，降低抄录和对时错误；
- 每个频率自动聚合为一个数据点；
- 标尺换算、表格和图表在同一界面完成；
- 浏览器直接运行，无需安装专业视频软件；
- 轨迹和重心可视化便于及时发现追踪失败。

这些是效率和一致性改进，不代表计量精度已优于人工分析。必须按 [验证协议](docs/VALIDATION_PROTOCOL.md) 对照后才能作精度结论。

## 当前明确未实现

- FFT、频谱和谐波分析；
- 驱动—响应相位差；
- 数字滤波、卡尔曼滤波或离群点剔除；
- 振动物体真实机械振幅反演；
- 数据持久化和 CSV 导出；
- 摄像头时间戳与音频硬件输出的精密同步；
- 理论受迫振动参数拟合。
