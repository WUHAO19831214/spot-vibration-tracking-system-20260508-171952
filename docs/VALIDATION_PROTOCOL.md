# 验证协议

本文件定义建议的验证方法，不代表仓库已经完成验证。

## 验证设备

- 固定支架、稳定背景和标定板；
- 可编程位移台或可产生已知位移的替代装置；
- 独立频率计、示波器、麦克风或加速度传感器；
- 可锁定曝光、对焦和白平衡的摄像头；
- 原始视频或屏幕录制手段。

## 静态定位验证

将红色光斑放置于不少于 10 个已知位置，每点采集至少 100 帧。记录自动重心均值、帧间标准差、参考偏差、丢失率和误识别率。

## 标尺换算验证

1. 在光斑平面放置已知标尺；
2. 分别标定 0.5、1、2、5 cm；
3. 比较估计长度与真实长度；
4. 在画面中心和四角重复；
5. 改变标尺深度评估非共面误差。

验收指标应在实验前规定。本仓库尚无经验证的精度阈值。

## 动态位移验证

用已知峰—峰位移驱动红色目标，覆盖多个位移、频率、亮度、分辨率和帧率。计算：

```text
bias = mean(Δy_measured - Δy_reference)
MAE  = mean(|Δy_measured - Δy_reference|)
RMSE = sqrt(mean((Δy_measured - Δy_reference)^2))
```

同时报告置信区间或一致性界限。

## 频率链路验证

程序频率来自 Web Audio 的设定值。应独立测量实际输出，比较设定频率与实测频率，并检查切换后的稳定时间。普通扬声器不能可靠输出接近 0 Hz 的声学驱动，默认 0–20 Hz 范围必须针对具体执行器验证。

## 自动扫频验收

建议设置 2–6 Hz、步进 1 Hz、停留 3 s：

1. 每个唯一频率只出现一行；
2. 切换前写入当前窗口的 `y_max-y_min`；
3. 停止扫频结算当前窗口；
4. 反向再次经过同频率不新增行；
5. 同频率多次结果保留较大 `Δy`；
6. 未锁定光斑时不产生虚假点。

## 人工视频分析对照

至少选取 30 个覆盖小、中、大位移的窗口，由两名分析者在不知道自动结果的条件下逐帧标注极值。比较自动与人工 `y_min/y_max/Δy`、分析时间、分析者间一致性和失败类型。

完成该对照前，只能声称系统减少操作步骤和自动完成数据关联，不能声称精度优于人工方法。
