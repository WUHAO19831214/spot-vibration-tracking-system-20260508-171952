# 研究设计

## 研究问题

1. 浏览器端颜色追踪能否稳定提取反射红色光斑的纵向位置？
2. 自动扫频条件下，光斑上下位移范围 `Δy` 如何随驱动力频率 `f` 变化？
3. 与人工逐帧标点相比，自动方法能否降低操作时间和记录差错？
4. 实时轨迹、表格和曲线能否帮助学生区分直接测量量与推断量？

## 操作性定义

- **自变量**：程序设置的驱动力频率 `f / Hz`。
- **直接观测量**：摄像头像素坐标中的红色光斑重心 `y / pixel`。
- **派生量**：频率停留窗口内的 `Δy_pixel = y_max-y_min`。
- **换算量**：`Δy_cm = Δy_pixel × cm_per_pixel`。
- **非直接测量量**：振动物体本体的机械振幅、速度、加速度、相位差和阻尼比。

## 数据流程

```text
Web Audio 设定频率
        ↓
频率停留窗口开始
        ↓
摄像头帧 → 红色阈值 → 加权重心 y(t)
        ↓
窗口内累计 y_min 与 y_max
        ↓
Δy_pixel = y_max - y_min
        ↓
1 cm 标尺换算为 Δy_cm
        ↓
每个唯一频率保留一条记录 → 散点图/视觉平滑连接
```

## 自动扫频

默认参数为起始 0 Hz、峰值 20 Hz、步进 1 Hz、每点停留 5 s。程序到达峰值后反向扫频；再次经过同一频率时保留较大的 `Δy`。

设定频率不等于装置实际获得的稳定正弦驱动力。普通扬声器在极低频段的实际输出必须用独立传感器确认。

## 建议的对照实验

### 追踪准确度

对同一段画面比较自动 `y_auto(t)` 与两名人工分析者的 `y_manual(t)`，报告 MAE、RMSE、最大误差、相关系数和人工分析者间一致性。

### 位移范围准确度

```text
error_Δy = Δy_auto - Δy_reference
relative_error = error_Δy / Δy_reference
```

参考值可来自人工逐帧标点或经过标定的运动平台。

### 教学效果

可比较实验完成时间、记录错误率、对直接/间接测量量的理解，以及频率响应曲线解释质量。

## 重复与报告

建议每个频率至少重复 3 次，同时记录升频/降频方向。论文应披露摄像头型号、分辨率、实测帧率、浏览器、光斑条件、标尺像素长度、光路结构、驱动设备、扫频参数、重复次数、排除标准、不确定度和软件 commit。

当前程序同频率只保存最大范围，正式研究若需重复统计，应同步保存原始视频或另行记录每次结果。
